MMA PARALLELISM & OCCUPANCY TRADEOFFS:
  • Ampere GPU: 108 SMs, 64 warps/SM max
  • MMA instruction: 16×8×16 tensor core operation per warp
  • Thread count determines warp count: warps = threads ÷ 32
  • MMA tile M-dimension: mma_M = warps × 16

  • Kernel constraint: (m_block×2) % threads == 0 must hold

KEY INSIGHTS:
  • Thread count is a fundamental architectural parameter

  • More warps = more MMA parallelism but lower occupancy
  • Constraint violations reveal hardware-software alignment requirements
  • This is where CuTe DSL exposes GPU microarchitecture details
