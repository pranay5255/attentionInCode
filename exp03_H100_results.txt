CONCEPT:
  CuTe DSL tiles the MMA as (num_threads//32, 1, 1) warps.
  Each warp executes an Ampere MMA instruction (16×8×16).

  More warps → larger MMA tile, more parallelism per CTA,

HARDWARE-SOFTWARE THREAD COUNT ANALYSIS:
================================================================================
MMA PARALLELISM & OCCUPANCY TRADEOFFS:
  • Hopper GPU: 132 SMs, 64 warps/SM max
  • MMA instruction: 16×8×16 tensor core operation per warp
  • Thread count determines warp count: warps = threads ÷ 32
  • MMA tile M-dimension: mma_M = warps × 16

CONSTRAINT ANALYSIS:
  • Kernel constraint: (m_block×2) % threads == 0 must hold
  • Violated constraints are valuable data points for optimization
  • 64 threads (2 warps): occ=1.00, 41.96 TFLOPS
  • 128 threads (4 warps): occ=1.00, 374.86 TFLOPS
  • 256 threads (8 warps): occ=0.50, 302.65 TFLOPS

KEY INSIGHTS:
  • Thread count is a fundamental architectural parameter
  • Each warp gets dedicated tensor core MMA operations
  • More warps = more MMA parallelism but lower occupancy
  • Constraint violations reveal hardware-software alignment requirements
  • This is where CuTe DSL exposes GPU microarchitecture details
